### 3. 알람 (SSE 데모)

- 설명: 팀 프로젝트 당시 chunk 에러로 구현 실패. 원인 정리와 성공 기준을 정의하고, 로컬 SSE + 목 환경으로 데모를 구현.
- 포인트
  - 서버 헤더: text/event-stream, Cache-Control: no-cache, 25s heartbeat
  - 클라이언트: 이벤트 수신 시 invalidateQueries(['alarms'])로 자동 반영
  - 성공 기준: 지속 연결, heartbeat 수신, 이벤트 후 목록/배지 갱신
  - 상세: SSE 회고/개선안([링크])

SSE: 시퀀스만으로 충분하며, 운영 포인트(재연결·백오프·리플레이·중복방지)를 꼭 표기.

--- 필수 기입: 구현 실패 기능 -> 오류/원인 파악, 성공 기준 정의/수정방식 설계
